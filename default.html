Large Language Model

LLM = Large Language Model

by Charles Iliya Krempeaux

A Large Language Model (LLM) is a type of machine learning and generative artificial intelligence (AI) model that is usually based on the deep learning transformer architecture, that is trained on large amounts of text to predict the next token in a sequence.

Some tasks and use-cases an LLM can be used to do are: text generation, natural language "understanding", question answering, summarization, paraphrasing, translation, source-code generation, source-code debugging, chatbots, text classification, text clustering, and reasoning-like behaviors without being explicitly programmed for each task (beyond the initial trainings).

At least at the time of writing, most people have probably experienced LLMs via AI) chatbots and agentic computing.

LLM Architectures

There isn't just one type of LLM. There are a number of different LLM architectures.

Some of the different LLM architectures are:

Autoregressive (Decoder-only) Models
These types of LLMs are good at generating text by predicting the next word in a sequence, one by one. These types of LLMs are useful for generating text. Examples of this type of LLM are GPT and LLaMA.
Autoencoding (Encoder-only) Models
These types of LLMs are designed to "understand" and analyze text (rather than generate text from scratch). These types of LLMs are useful for classification or sentiment analysis. An example of this type of LLM is: BERT.
Sequence-to-Sequence (Encoder-Decoder) Models
These types of LLMs combines both encoding and decoding. An example of this type of LLM is: T5.

Pre-Training versus Post-Training

Sometimes you will hear people talk about pre-trained models versus post-trained models, or pre-training versus post-training> — usually in the context of how LLMs are made.

These terms don't seem to be used consistently across the industry, but — here is the general idea —

The phrase “pre-trained model” usually refers to the “base model”. Where the phrase “post-trained model” usually refers to what you get by modifying the “base model” in various ways. (Of course, you are probably now wonder, what is a “base model”.)

Pre-Training

An LLM starts of as a large Artificial Neural Network. (One of the things that the word “large” in “large language model” refers to is how large the Artificial Neural Network is.)

This is an Artificial Neural Network that probably hasn't learned anything yet. It weights/parameters may be randomized.

An Artificial Neural Network, or any other type of Machine Learning, Artificial Intelligence, or Statistical model, with random weights/parameters generally isn't useful yet. To make it useful, it needs to be taught. The process of teaching it is usually called training.

For LLMs, this pre-training process tends to be a giant optimization process to get the LLM to predict the next token (i.e., fill in the blank) of things like these:

  • “The color of the apple is _____”
  • “The capital of Iran is _____”
  • “How much wood could a woodchuck _____”
  • etc.

This process usually involves running and using thousands of GPUs, perhaps spending tens of millions of dollars, and trained a brand new architecture from scratch (or a massive new compute iteration) on raw data.

Most people don't use these types of raw pre-trained models. A raw pre-trained model does not know it is an AI assistant. A raw pre-trained model does not know how to follow instructions. If you give a raw pre-trained model a prompt such as:

🠶 Write a Golang function to reverse a linked list.

A raw pre-trained model might not write the source-code. Instead, a raw pre-trained model might, for example, output a list of other homework questions because it thinks you are pasting a computer science syllabus.

But, this is this raw pre-trained model is the base model. In some ways, you can think of it is as a chaotic, autocomplete engine.

The base model is usually: weird, inconsistent, difficult to control, not very helpful. The base model can complete text (such as “The color of the apple is _____”) but may not behave like an assistant.

Post-Training

Post-training is everything done after we have this raw base model.

Post-Training takes this raw pre-trained model and tries to get it to behave in a more useful and more desirable way (from the point of view of the people doing the post-training).

There is more than one type of post-training:

  • Supervised Fine-Tuning (SFT)
  • Reinforcement Learning from Human/AI Feedback (RLHF) (RLAIF)
  • Reasoning/Chain-of-Thought (CoT) Alignment

Supervised Fine-Tuning (SFT)

One type of post-training is: supervised fine-tuning (SFT).

The supervised fine-tuning phase is where a model is transformed into an AI assistant.

With supervised fine-tuning, the model is trained on a set of high-quality, curated [Instruction] -> [Response] pairs. For example:

This teaches the model the format of being an assistant (e.g., "When a user asks a question, reply directly and politely").

For more information on this, see: Supervised Fine-Tuning.

Reinforcement Learning from Human/AI Feedback (RLHF) (RLAIF)

Another type of post-training is: reinforcement learning from human/AI feedback.

Reasoning/Chain-of-Thought (CoT) Alignment

Yet another type of post-training is: reasoning/chain-of-thought.
